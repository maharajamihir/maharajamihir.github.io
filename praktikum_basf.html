3. Internship

3.1. Expectation

  • Expected it to be similar to own projects/experienced projects

3.2. Reality

  • Way bigger code than expected or experienced!
  • Big companies value security a lot
  • Working in a bigger team requires a lot more coordination
  • Daily standup meetings and updates are important
  • Despite flexible working hours, a fix schedule is important especially for coordination with teammates and planning by SCUM-master
  • When stuck on an issue, dont hesitate to ask for assistance by a more senior developer
    • In past projects I learned that I should solve problems myself
    • University also conveys the “solve it yourself”/“do it yourself” mentality
    • But company needs quicker issue handling and there is a closer team, so asking for assistance is more efficient
  • Bigger CI/CD pipeline needed, manual deployment not possible everytime
  • Multiple environments rather than one single one
  • Companies focus on detail a lot. Product has to be perfect rather than quickly implemented. Example: Frontend has to look exactly like design. Developer doesn’t have a lot of freedom here but has clear instructions.
  • Since not everyone is a developer there is a clearer role distribution
  • Main focus is not learning something new but deliverying as fast as possible

  • Planning, engineering and communication is mostly more important than the coding itself

4.2. Aquired skills

  • frontend development with the React framework
  • AWS overview

  • Team skills
  • Communication skills

5. Problems I faced

  • Was used to coding whenever I wanted. Due to working times of others I have to make fix schedule
  • Codebase bigger than expected. Dive into code was difficult in beginning
    • Reached out, asked for a code overview and inspected the frontend in the browser and was able to reverse track the components code.
  • BASF Laptop didnt have Admin rights and it wasnt well documented how to aquire those
    • Fix:
      • Contacted several people how to fix it
      • Called IT support several times
      • Access for local admin rights here: https://my-link.basf.com/en/tips/03014041#tip-2218
      • Fixed myself by accessing “LocalAdminManager”
        • Here you switch to Local Admin in bottom left corner
        • Choose the application you need admin rights for (mostly command shell or file explorer)
        • execute an installer from file explorer or command shell, which is opened with admin rights
        • This way it wont even ask for admin rights while installing
      • Not sure if this only worked because I had requested an INST admin account or if this works for everyone. If it does, it is a security breach.
